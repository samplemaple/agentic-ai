## 一、项目定位

### 内容方向
AI 技术 + AI 社会话题，两种风格混合：

| 类型 | 风格参考 | 占比 | 说明 |
|------|---------|------|------|
| **技术资讯解读** | 赛博禅心 | ~60% | 海外 AI 新闻/产品更新的中文解读，面向开发者 |
| **AI 社会话题** | 大张的自留地 | ~40% | AI 对生活的影响，通俗易懂，面向普通人 |

### 每日产出
- 3 篇文章
- 技术解读 2 篇 + 社会话题 1 篇（大致比例，按当天热点灵活调整）

### 发布流程
```
Agent 每天 8:00 生成 3 篇文章
    ↓
写入 /tmp/wechat-articles/
    ↓
8:10 通过邮件发送给我
    ↓
我审核、修改
    ↓
手动发布到公众号
```

> 后续公众号认证后，可切换为 API 自动存草稿模式

## 二、内容风格规范

### 技术资讯解读（参考赛博禅心）

- 语气：冷静克制，不带情绪，像技术文档的口语版
- 结构：功能点逐个拆解，场景一二三
- 标题：直述核心信息，不搞悬念（如"Codex 凌晨更新，将屏幕内容放进记忆"）
- 字数：1500-2500 字
- 特色：把英文官方文档/博客翻译+重组，加自己的理解
- 配图说明：标注"此处配官方截图"或"此处配产品界面截图"

### AI 社会话题（参考大张的自留地）

- 语气：口语化、感性、有情绪
- 结构：故事引入 → 热搜/事件 → 案例分析 → 观点总结
- 标题：情绪驱动，制造悬念（如"张雪峰被复活，越想越觉得后背发凉"）
- 字数：800-1500 字
- 特色：用普通人能懂的语言讲 AI 话题，重观点不重技术
- 配图说明：标注"此处配相关截图"或"此处配 AI 生成的示意图"

### 通用规范

- 段落短，每段 2-4 句话，适合手机阅读
- 不用 markdown 格式（公众号不渲染），用纯文本 + 空行分段
- 代码块用等宽字体标注（Agent 输出时用 ``` 包裹，发布时手动调格式）
- 文末加引导语（如"你怎么看？欢迎留言讨论"）

| 模型 | deepseek-chat（生成）/ deepseek-reasoner（复杂分析） |
